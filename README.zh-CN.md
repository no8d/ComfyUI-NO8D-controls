# ComfyUI-NO8D-controls

[English](./README.md) | 简体中文

ComfyUI-NO8D-controls 是一套面向实际出图迭代的 ComfyUI 自定义节点。它把 LoRA 组合控制、提示词扩写、图像载入、带遮罩绘制的生成节点、A/B 对比、空 latent 创建和图文数据集保存整合到一套工作流里。

项目原则是：能用 ComfyUI 原生机制就优先用原生机制，包括节点执行、列表输入、图像预览、右键菜单、队列和图结构展开。只有在原生控件无法满足工作流效率时，才写轻量自定义前端 UI。

![ComfyUI-NO8D-controls](docs/images/no8d-control-banner-readme.jpg)

## 节点列表

节点位于 `NO8D-control` 或 `NO8D-controls` 分类。

- `NO8D-LoRA 堆栈`
- `NO8D-提示词`
- `NO8D-提示词预览`
- `NO8D-图像载入`
- `NO8D-生成`
- `NO8D-A/B 对比`
- `NO8D-图文保存`
- `NO8D-空 latent`

## 安装

把仓库克隆到 ComfyUI 的 `custom_nodes` 目录：

```bash
cd ComfyUI/custom_nodes
git clone https://github.com/no8d/ComfyUI-NO8D-controls.git
```

安装后重启 ComfyUI，并在浏览器中强制刷新页面。不需要额外的前端构建步骤。

## 常见连接方式

```text
Checkpoint Loader
  MODEL -> NO8D-LoRA 堆栈 -> NO8D-生成 -> IMAGE
  CLIP  -> 提示词编码链路      -> NO8D-生成
  VAE   ---------------------> NO8D-生成
  latent --------------------> NO8D-生成

NO8D-生成 -> NO8D-A/B 对比
NO8D-图像载入 -> NO8D-提示词 -> NO8D-提示词预览 / NO8D-图文保存
```

项目内置了一个 ComfyUI 示例工作流：[examples/NO8D-controls-example.json](examples/NO8D-controls-example.json)。

这个示例保留节点布局和连接关系，但已经移除本地图像选择、私有 LoRA 路径、临时预览图片和本地保存路径，方便公开分享。导入后请替换为自己的图像和 LoRA。

## 节点说明

### NO8D-LoRA 堆栈

用于在一个节点中组合多个 LoRA，并输出应用 LoRA 后的模型。

![NO8D-LoRA 堆栈](docs/images/stack-node-readme.jpg)

- 在一个节点里添加、删除、启用、关闭和排序多个 LoRA。
- 用滑条或数字框调整每个 LoRA 的权重。
- 每个 LoRA 可以设置独立的最小值和最大值。
- 每个 LoRA 可以填写触发词。启用且权重非 0 的 LoRA 会把触发词合并为一个字符串输出。
- 数字框键盘调参规则接近 ComfyUI：方向键小步进，Shift + 方向键大步进。

### NO8D-提示词

通过配置好的提示词 API，把简短想法、参考图像，或两者一起整理成完整正向提示词。

![NO8D-提示词](docs/images/prompt-plus-node.png)

三种输入情况：

- 只输入文本：在 `输入文本 / Input text` 中写想法，节点会扩写为完整图像提示词。
- 只接入图像：节点会根据图像反推可用提示词。
- 文本 + 图像：文本代表用户意图，图像作为视觉参考；两者冲突时优先遵守文本。

`风格` 提供 `自行判断` 以及四个大分类：`写实摄影`、`动漫插图`、`手绘艺术`、`数字艺术`。四个分类不是固定的具体风格，而是限制模型选择范围：模型会对照参考图或简短文本，在所选分类内判断一个最合适的子类，并把子类及其可见特征明确写入提示词。例如，`写实摄影` 会根据证据选择手机随拍、纪实/街拍、单反或无反人像、商业棚拍、建筑摄影、影视剧照等，而不会统一套用某一种摄影质感；`手绘艺术` 会进一步判断油画、水彩、水粉、素描、炭笔、版画等实际媒介。`自行判断` 同样只会先从这四个大类中选择一个，再判断其具体子类，不会使用 `氛围摄影`、`电影感风格` 等宽泛词代替子类。模型不会罗列多个候选子类。旧工作流中的细分风格会自动迁移到最接近的新分类。

图像反推会分别识别并明确写出主体景别、仰俯机位、相机在主体前后左右的方位、镜头朝向、主体自身朝向、主体在画面中的左右/上下位置、镜头透视/视场、画面倾斜和空间纵深。图像输入的最终提示词首句必须同时包含具体风格子类和镜头—主体空间关系，明确相机相对主体的高低与观看方向、正面/侧面/背面方位、主体在画面中的位置和前后景层次，以及镜头类型带来的透视效果；多主体画面还必须说明各主体的左右和前后关系。即使没有填写任何相机相关文本，模型也必须从图像判断这些信息；仅复述用户补充的镜头词不算完成图像分析，也不能用“动态角度”“沉浸式构图”“电影感构图”等泛词代替。相机方位、主体朝向和画面位置使用独立坐标，不会把“相机在主体右侧”“主体朝向画面右侧”和“主体位于画面右侧”混为一谈；该规则同样适用于建筑、车辆、产品、动物和其他非人物主体。`主体景别` 设为 `自行判断` 时会保持参考图的实际景别；手动选择其他景别时，只改变主体远近，仍尽量保留参考图中的机位和镜头特征。

镜头—主体关系是每次输出的必需内容，不限于图像反推。只输入文本时，模型会根据主体、动作、情绪和风格主动设计明确的景别、机位高低、观看方向、相机方位、主体画面位置、空间层次和镜头透视；文本与图像同时输入时，文本中的明确要求优先，其余关系遵循图像证据。动漫、手绘艺术和数字艺术同样必须描述构成画面的虚拟镜头或观看视点，不能因为不是摄影而省略。

自然语言模式在单次 API 请求内填写一份结构化视觉分析，包括主体头部/前端、重心、腿部/后端三个关键锚点、归一化框位、主体主轴、画面占比、可见身体范围、机位高低、观看方向、主体方位、镜头类别、透视证据及每个次要主体的位置和层次，并另外提供不含镜头构图术语的场景描述。节点依据分析数据生成最终提示词并移除内部结构，最终仍只向下游输出自然语言提示词。标准/详细模式的最终提示词上限仍分别为 256/512 tokens；请求会额外预留少量隐藏分析空间，不占用最终提示词额度。

为避免模型在最终文案中推翻自己的坐标分析，节点会根据结构化数据确定性生成风格与构图句：由可见身体范围计算景别，由关键锚点计算主体主轴和画面跨度，由镜头类别生成对应透视描述，再与模型提供的场景细节合并。小于目标尺寸的参考图会先按比例放大到长边 768 像素，图像分析温度最高为 0.1。Thinking 视觉模型通常明显慢于同规模 Instruct 模型；需要较快、较稳定的图像反推时优先选择视觉 Instruct 模型。

API 管理器可分别设置 `文本模型` 和 `图像模型`：纯文本扩写使用文本模型，纯图像及文本+图像使用图像模型。旧配置如果选择了 `*-Thinking`，并且服务模型列表中存在同名 `*-Instruct`，会自动把后者保存为图像模型，同时保留原文本模型。API 读取超时后不会再把同一个慢请求完整重试一次，避免一次失败等待两个 120 秒周期。

`长度` 的标准模式目标为约 256 tokens，API 输出硬上限为 256；详细模式目标为约 512 tokens，API 输出硬上限为 512。两种模式都会尽量接近目标长度但不允许超过上限，并优先输出主体、动作、相机与主体关系、画面位置和构图，再补充风格、材质与气氛，降低下游文本编码器截断关键空间信息的风险。

`固定提示词 / Fixed prompt` 用于固定前缀，例如 LoRA 触发词。它会自动追加到生成提示词前面。

### NO8D-提示词预览

显示提示词文本，支持手动编辑，并可把编辑后的文本发送到后续节点。

- 上游提示词非空时，编辑窗口始终显示最新上游提示词，与 `auto` 开关无关；上游为空时保留编辑窗口原内容。
- `auto` 开启：普通运行时，上游非空则输出上游提示词；上游为空则输出编辑窗口内容。
- `auto` 关闭：普通运行时输出空字符串，但仍按上一条规则更新或保留编辑窗口。
- `发送`：忽略 `auto` 和上游状态，使用编辑窗口内容单独运行下游；编辑窗口为空时输出空字符串。

![NO8D-提示词预览](docs/images/prompt-view-node.png)

### NO8D-图像载入

用于把一张或多张本地图像载入工作流。

![NO8D-图像载入](docs/images/load-images-node.png)

- 支持点击文件夹按钮选择图像、拖拽图像到节点，也可以点击预览区后按 `Ctrl+V` 粘贴复制的图像。
- 支持单选和多选。
- 每张缩略图右上角提供眼睛角标，可单独启用或关闭该图像的节点输出。
- 当候选图片全部关闭时，节点返回空图像列表，下游 list 执行会跳过。
- 支持拖动缩略图排序。
- 双击缩略图会打开原图，不再触发工作流或单图输出。
- 队列运行时：如果有选中图像，则输出选中的图像；如果没有选中图像，则输出全部载入图像。
- 输出采用 ComfyUI list 方式，让下游节点可以按“算完一张再算下一张”的方式处理。

### NO8D-生成

把 ComfyUI 采样流程封装为一个紧凑生成节点，并提供可编辑的图像/遮罩预览。

![NO8D-生成](docs/images/generate-node.png)

- 控制采样器、调度器、步数、CFG、降噪和随机种子。
- 支持种子锁定或随机。
- 支持画笔、套索、橡皮擦、遮罩羽化、透明度、颜色、反转和清除。
- 工作流会保存可编辑遮罩笔画；清除遮罩或继续编辑后，旧上传任务不会回写当前状态。
- 激活遮罩工具或画布中保留遮罩时，会自动进入 ComfyUI 原生差分扩散局部重绘，不需要额外的模式下拉菜单。
- 使用原生 `VAEEncodeForInpaint` 与 `LatentBlend` 随 denoise 平滑增强遮罩区变化；denoise 1 时保留 30% 原图 latent、混入 70% 清除 latent，以提高实心区变化幅度并保留必要连续性。
- 画布继续以清晰分区显示羽化范围，后台采样和最终合成则使用独立的 32 级线性渐变软遮罩，以减少重绘接缝。
- 同一次遮罩会话固定使用原底图：随机种子会从同一底图生成可比较的候选，锁定种子可稳定命中缓存；清空遮罩后重新绘制，才会从当前预览开始新的底图会话。
- 只输出最终图像；有意不输出 latent 和 mask，避免图像空间合成后的下游语义产生误解。
- 内部使用 ComfyUI `GraphBuilder` 展开为原生采样/解码节点，因此可以正确处理 list 输入。
- 尽量保留 ComfyUI 原生图像状态，方便右键菜单等原生行为。

### NO8D-A/B 对比

用于对比两路图像。

![NO8D-A/B 对比](docs/images/ab-preview-node.png)

- `image_a` 显示在左侧。
- `image_b` 显示在右侧。
- 对比图像下方提供独立信息栏，分别以 `A · 宽 × 高`、`B · 宽 × 高` 显示当前图像的原始尺寸，不再覆盖图像内容。
- 提供 `image_a` 图像输出接口；开启“自动输出图像 A”时将当前 `image_a` 传递到下游，关闭时安全阻止该下游分支，同时不影响节点自身的对比预览。
- 如果只接入一路图像，缺失的一侧显示该路上一次运行的图像；第一次运行没有历史图时显示空白。
- 接收到图像列表时，右下角页码可点击切换对比页。

### NO8D-图文保存

用于保存图像和文本，适合整理图文数据集。

![NO8D-图文保存](docs/images/save-node.png)

- 文件名规则支持固定文本、原文件名、日期时间和尺寸等级。
- 命名规则可以拖动排序。
- 可把提示词或 caption 与图像一起保存。

### NO8D-空 latent

按常见模型类型、比例和短边尺寸创建空 latent。

![NO8D-空 latent](docs/images/empty-latent-node-readme.jpg)

- 支持 SD/SDXL、SD3/Flux/Krea2、Flux2 等尺寸预设。
- 支持常见竖图比例和反转后的横图比例。
- 输出 latent，同时输出计算后的宽度和高度。

## 提示词 API

Prompt 节点使用 NO8D 提示词设置中的 API 配置，支持 OpenAI 兼容 API 或本地 LLM 兼容接口。

可在设置中完成：

- 配置 API 服务；
- 验证可用模型；
- 选择默认提示词 API；
- 编辑提示词撰写规则。

API key 和本地配置属于本地环境，不要提交到仓库。

## 语种自适应

前端会从 ComfyUI 设置、浏览器 localStorage、页面语言和可见 ComfyUI 文案中判断当前语言。当前支持英文和简体中文。

节点 UI 会在启动时初始化语言，并在浏览器 `storage` 和 `languagechange` 事件发生时刷新标签。项目已经移除周期性语言轮询。

## 开发说明

- Python 节点通过 `NODE_CLASS_MAPPINGS` 注册到 ComfyUI。
- 前端扩展位于 `web/`，由 `WEB_DIRECTORY = "./web"` 加载。
- `NO8D-Generate` 通过 `GraphBuilder` 展开为 ComfyUI 原生采样/解码节点。
- `NO8D-图像载入` 和 `NO8D-Generate` 以 ComfyUI list 执行为核心，不做手写批量循环。
- 后续开发应继续优先使用 ComfyUI 原生行为，避免重复实现预览、队列和图执行机制。
